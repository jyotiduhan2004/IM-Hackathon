## Priority stack — three agents agreed

### Tier 1 (do first, 3-5h total, compounding)

1. **Second-pass stub-filler** (~2h)
   - All 3 audits flagged this. Stub entities never get backfilled; `lucky-agarwal`,
     `wazuh-mcp`, many others are empty scaffolds despite their sources being on disk.
   - Build `scripts/backfill_stubs.py` that walks every stub (`last_compiled: "stub"`
     or empty `sources:[]`), grep-searches `raw/*.md` for mentions, rebuilds the page
     with the full source list, then triggers a small compile pass on just those.
   - Wire into `make pipeline` so it runs at end of every compile.

2. **Promote broken-wikilink to hard error** (~30 min)
   - DONE in this session for `validate_wiki.py` — need to commit + verify the big
     failure doesn't mask important bugs.
   - Follow-up: `create_missing_stubs` in lint already exists — run it in the
     pipeline so new broken links get stub pages.

3. **Verbatim bug/ticket table preservation** (~2h)
   - Audit 2 called this out explicitly (dropped 12-row bug matrix from
     `dynamic-smart-rfq-form`, 75% of TrustPulse content). Loses the most
     actionable signal in the wiki.
   - Detect tables in source body with regex (`^\|.+\|\s*$` for 2+ lines)
   - Prompt rule: if email has a table, emit a markdown table in the wiki page
     with EVERY row intact
   - Validator check: if source has a table with N rows, wiki page must have a
     table with ≥N rows. Flag as warning otherwise.

### Tier 2 (afternoon, if Tier 1 lands clean)

4. **Trivial-message filter** (~3h)
   - Deterministic regex at ingest for obvious acks (`+1`, `thanks`, `lgtm`, <20
     words with `in_reply_to`)
   - 40-60% reduction in compile calls on mailing lists
   - Level 2 (cheap classifier) deferred

5. **compile_parallel.py per-batch timeout + file lock** (~1.5h)
   - `asyncio.wait_for` around each agent invocation (max 10 min/batch)
   - `fcntl.flock` on shared entity pages (per-filename lock) to prevent last-
     write-wins data loss with concurrency > 1
   - Prereq for any overnight parallel run

6. **log_insight tool** (~1h)
   - New tool the agent calls optionally to emit meta-commentary
   - Writes to `docs/insights/YYYY-MM-DD.md`
   - User clarified: emit raw doubts even without a proposed action

### Tier 3 (evening / tomorrow, research-driven)

7. **Study atomicmemory/llm-wiki-compiler's two-phase pattern**
   - Phase A: extract all entities/concepts from a batch (no writes)
   - Phase B: write pages using the extracted graph as input
   - Reportedly "strictly better" than per-thread compile. Evaluate before
     re-architecting.

8. **Upgrade to Deep Agents v0.5** (`AsyncSubAgent`)

   - Built-in subagent management, proper state isolation

9. **CompositeBackend** (raw/ read-only, wiki/ write-allowed)
   - Eliminates entire class of "agent accidentally wrote to raw/" bugs
   - From Deep Agents docs (agent scan found it)
