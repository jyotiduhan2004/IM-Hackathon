## Recommendations

1. **Compiler prompt**: require the `sources:` field to include every raw file whose To/From/CC/body references the entity; never emit stubs for names present in at least one raw file.
2. **Enforce page_type invariants**: if `email: <x>@indiamart.com` is present in frontmatter / body → must be `entity`. If slug ends in `-team`, `-bot`, `.com`, `.net` or matches a mailing-list address → must be `system`. Add a post-compile lint.
3. **Dedup check**: after compile, hash page bodies (excluding `last_compiled`); collision → fail the build. Would have caught the `export-indiamart`/`tawk-to` bug.
4. **Preserve technical depth**: current compiler drops root-cause, ticket IDs, bug summaries, and test counts in favour of prose abstractions. Either keep a "Technical Details" section verbatim from source or add a `raw_excerpts:` block.
5. **Fix the trailing-block rendering bug** (see `neeraj-agrawal.md` lines 60-66) — looks like a chunk concatenation issue.
6. **Role/title capture**: ingest email signature blocks to populate a `role:` field on entity pages (would fill in "Lead Engineer - Cybersecurity Team" for Kajal, etc.).
