# NORTH-STAR Proposal — Decisions Log

> **The wiki end-state spec** lives at [`docs/NORTH-STAR.md`](../NORTH-STAR.md).

> This file is the *historical record* of the 2026-04-15 Q&A pass that produced the spec — why each decision was made, not what was decided.

## How we got here

Before 2026-04-15, the project had 6 competing north-star statements across README / `docs/issues/{07..11}` / `BACKLOG.md`, and 7 unresolved binary decisions. The wiki corpus audit showed ~80% of pages read like a filing cabinet (entity pages at 0%, systems at 5%, topics at 40%).

A Q&A pass with the project owner ratified the direction: **compiled, curated Wikipedia for IndiaMART, company-wide from day 1, no manual edits, progressive disclosure, concept pages not per-event**.

Supporting research: [`research/01-reconciliation-report.md`](research/01-reconciliation-report.md), [`research/02-codex-contributions-audit.md`](research/02-codex-contributions-audit.md), [`research/03-wiki-reader-experience-audit.md`](research/03-wiki-reader-experience-audit.md).

## Decisions log (2026-04-15 Q&A pass)

1. **Audience**: company-wide, day 1 (not owner-pilot)
2. **Live ingestion**: defer to Phase 2; manual re-runs fine for now
3. **Mailing lists**: single (`marketplacelaunch@indiamart.com`) for now
4. **Codex collaboration model**: out of this proposal's scope (decided elsewhere)
5. **6-month-new-joiner test**: real acceptance bar (not editorial guidance)
6. **Entity pages**: hidden from primary nav; reference-only (lazy creation on wikilink)
7. **Page types**: drop `timeline` + `conflict` (zero pages, no use case); 4 visible + 2 reference
8. **Structure**: domain-first + decisions + tags + reference people
9. **Edit model**: read-only / compiler-only (glossary is auto-gen, documented exception)
10. **Domain hubs**: 8 validated against corpus (see NORTH-STAR §"The 8 domain hubs"); compiler-generated rollups with multi-tag topics + periodic meta-agent for evolution (Phase 2)
11. **Topics**: concept pages, not per-thread (wikipedia model — "Seller ISQ" is one page that grows)
12. **Sub-initiatives**: sections in parent page (Phase 1); multi-level hierarchy (Phase 2)
13. **Decisions**: lazy creation (only when a topic page wikilinks to one), but compiler actively scans emails for decision-shaped content when updating topics. Bar: meaningful change (e.g. 5%→50%, deprecation), not trivial ack
14. **Topic dedupe**: permissive compile + separate dedupe agent (merge near-dups post-compile)
15. **Glossary**: auto-generated by compiler
16. **Page template**: loose guidance + sensible defaults, no hard validation; compiler may deviate with reason
17. **Home page**: 2-sentence intro + 8 domain cards + recent activity ribbon + search + glossary card
18. **Status field**: `active / superseded / archived` with viewer badge; compiler sets supersession from email language
19. **Pipeline orchestration**: one unified `make pipeline` command
20. **Trivial-message filter**: yes (skip acks, calendar invites, one-liners — addresses ~30-50% of traffic)
21. **Synthesis self-review**: yes — extend PR #73's `check_my_work` pattern with a synthesis critique; rewrite if the page is a filing cabinet instead of synthesis

## Open questions still worth asking

These weren't blockers to ratifying the spec but affect implementation choices (see BACKLOG):

- Source rendering verbosity (current: collapsed details + 2,500-char excerpt per email; concern: topic pages with 100+ sources over time)
- People page lifecycle (when archived? auto-prune inactive?)
- Tag vocabulary (free vs constrained; currently permissive + dedupe-normalizes)
- Dedupe agent cadence (every compile run, or less frequent to save cost?)
- How domain rollups handle the ~10% of topics that span 0 or many domains

## What this proposal explicitly drops or defers

From the 2026-04-15 consolidation:

- 5 of the 6 pre-consolidation north-star statements → only `docs/NORTH-STAR.md` remains (other 5 archived)
- `timelines/` + `conflicts/` page types → dropped
- `navigation_role` frontmatter → not shipping (zero code references, zero pages have it)
- "Ownership History" required section → Phase 2 or cut
- "How the View Changed" required section → Phase 2 or cut
- Capability page middle tier (personas doc) → cut
- Promotion criteria tooling (personas doc) → cut
- Hand-written domain hubs → replaced by compiler-generated rollups
- Strict template validation → loose guidance only
- Owner-pilot framing → dropped (company-wide from day 1)

See [`docs/archive/README.md`](../archive/README.md) for what got moved and why.
