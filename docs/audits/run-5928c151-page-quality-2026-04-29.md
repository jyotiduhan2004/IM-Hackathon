**Fix**: deterministic post-batch hook that scans every modified page for `[^msg-XXX]` markers, looks up the canonical raw path (deterministic from message_id → `raw/...` is in Postgres), and either appends a `## References` section if missing or backfills missing defs. This is exactly the kind of work the user instructions name as a coordinator concern: "Anything driven by [stable identity] — compute it deterministically — never ask the LLM."

### 2. Pattern (problem): Past-tense narrative leads (scorer caught one, sample suggests more)

`soi-product-gst-screen-ab-test` got summary_currency=1/10. Quote: *"Final outcome: the new photo icon was adopted (without teal color), while the GST screen field reordering was reverted."* That's a closing-summary paragraph, not a current-state lead — even though the page is `status: active`. The Wikipedia-style framing is *"X is a Y that does Z; as of [date], it covers W"* — present tense with explicit recency anchor. Several other pages get this right (supply-chain-security, real-time-d-rank), so the prompt teaches it; the agent forgets when the experiment has a clean end-state.

### 3. Pattern (problem): Wikilink path-shape mismatch (`[[type/slug]]` vs validator's bare-slug)

The validator is comparing `[[system/lmshelp]]` against bare-slug index entries and missing the type prefix. This is a validator bug, not an agent bug — but the dipstick noise is misleading anyone reading "broken wikilinks" as "agent invented page names."

The 1 deep-read genuinely-broken case worth flagging: `[[topic/buylead-whatsapp-display]]` on `whatsapp-photo-buylead-optimization` — that page does **not** exist (`find ... -name 'buylead-whatsapp-display*'` is empty). So it's a mix of validator false-positives + real broken-link agent invention.

## Recommendations (ordered by leverage)

1. **Coordinator-side `## References` backfill.** The agent reliably emits `[^msg-XXX]` markers; it unreliably emits the defs. This is exactly the deterministic-identity case from CLAUDE.md ("if identity is stable... compute it deterministically — never ask the LLM"). For every modified wiki page, the coordinator should: (a) extract every `[^msg-XXX]` marker, (b) look up its raw path from the messages table (the markers are derived from the same hash), (c) if no `## References` section exists, append one; (d) for each missing def, append a definition line. This single hook would have moved 9 pages from "unverifiable" to "verified" in this wave.
2. **Validator: tolerate `[[type/slug]]` form.** The dipstick currently false-positives 6+ links per wave because it expects bare-slug. Either teach the validator the type-prefix form or have the agent emit bare slugs. (The user-instruction examples use `[[topic/seller-isq]]` form, so the validator should accept it.)
3. **Heuristic-scorer addition: footnote-def integrity.** Add a `score_footnote_integrity` heuristic that compares body `[^msg-...]` ref count to def count. Penalize pages with `body_refs > 0 and defs == 0`; severely penalize. The current scorer's `source_density=10/10` saturated on every one of these 9 broken pages, which is misleading.
4. **Past-tense lead detector.** A regex on the first sentence of body for terminal-aspect verbs (*was adopted*, *was reverted*, *concluded*, *final outcome*) and zero present-tense state could catch the soi-product-gst class. Cheap to add; would have caught the one bottom-1 page.
5. **Dedupe `## Related pages` + `## Related` body blocks deterministically.** 4 of 18 pages emitted both. Coordinator hook: collapse both into one canonical body section (or remove from body and rely on FM `related:`).
