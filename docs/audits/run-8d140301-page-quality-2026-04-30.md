## Top 5 patterns

### 1. Pattern (new defect): inline-colon-after-marker syntax mistake

The **dominant new failure mode** this round. Agent writes

> From [[X]] on 2026-02-25 [^msg-c3b9ff9a]:

mid-sentence to introduce a list, where the trailing `:` is meant as
"colon-then-list" but the linter sees `[^msg-XXX]:` and treats it as a
phantom def line. **32 occurrences across 39 pages**, of which **5 pages
have refs that have ZERO matching def lines** (the agent forgot to add a
real def in `## References`). Examples in
`bl-recommendation-grid-community-leads` (L38), `im-insta-two-click-
hotlead-journey` (L42), `product-scraping-agent-n8n` (L36), `search-
context-bl-enquiry-forms` (L100), `systems/rfq-display-service` (L41+L53).

**Why prior audit missed it**: ee90d7ce's F1 sweep treated `[^X]:` line-
starts as defs without checking whether `<path>` followed. The simple
sweep matches the validator and passes; the strict sweep catches it.

**Recommendation**: tighten the validator regex to "def line == column-0
`[^X]:` followed by a non-empty path/URL on the same line", and emit a
warning for `[^X]:` mid-sentence (the agent should write `On 2026-02-25
[^msg-X], ...` instead).

### 2. Pattern (corruption): empty frontmatter pages slip through

Two pages this run have `---\n{}\n---` frontmatter — fully empty.

- `topics/dynamic-district-visibility-on-bl-card.md` (9 lines, no body refs)
- `topics/user-search-context-bl-enquiry-enrichment-desktop.md` (19 lines, 3 unique refs, all missing defs)

These have valid filenames in `/tmp/run-8d140301-pages.txt` (so the
coordinator wrote *something*) but YAML frontmatter is blank — no title,
no status, no sources, no owner. **The validator/critique stage didn't
catch them.** Likely a partial agent abort + draft handoff issue.
mtime is fresh (2026-04-30 03:33 / 03:18). One scored 6.4 by the
heuristic (which has no frontmatter check), one scored 6.6.

**Recommendation**: coordinator-side guard — if a page is committed
with empty frontmatter, fail the batch and re-claim the message_ids.
This is exactly the "coordinators verify, LLMs propose" rule from
CLAUDE.md.

## Recommendations (ordered by leverage)

1. **Validator regex fix for inline-colon-after-marker** — change `def
   line` detection to require `^[^X]:\s+\S+` (column-0 marker followed
   by a non-empty path). Emit a warning when `[^X]:` appears mid-line.
   This catches the 7 cases this round and the 32 inline-colon hits.
   Trivial to land; covers the strict-sweep regression. **Highest ROI
   this round.**

2. **Empty-frontmatter coordinator guard** — if a written page has YAML
   frontmatter that parses to empty dict, fail the batch and re-claim
   the message_ids. The validator/critique should never let
   `dynamic-district-visibility-on-bl-card.md` and
   `user-search-context-bl-enquiry-enrichment-desktop.md` land.

3. **Coordinator-side staleness lint (F18)** — at compile time, scan
   touched pages for `## Recent changes` newest date; if `> 60 days` AND
   `status: active`, emit warning; if `> 120 days`, emit blocker. 26
   pages this round would flag — clear leverage.

4. **F2 prompt-by-example finally** — two rounds of #282 verb-only edits
   producing zero observable repair behavior is sufficient evidence to
   ship the worked-example boy-scout clause from ee90d7ce findings F2.
   The two-section `## Related` collapse example is concrete and won't
   regress greenfield pages.

5. **F12/F20 system-page archetype: prompt-by-example + lint combo** —
   add `indiamart-n8n-nodes` (from ee90d7ce) as the canonical system-
   page worked example in the prompt; add coordinator lint that flags
   any system page missing `## Recent changes` or `## Open questions`
   for forced rewrite. `export-indiamart-com` is the test case — if
   the next run's update doesn't refactor it, the lint isn't strict
   enough.
